## 什么是偏差日志

偏差日志（L5 Layer）是量化交易系统的第五层，用于记录和分析**系统决策与实际执行之间的偏差**。这是连接"理论"与"实践"的关键桥梁。

### 为什么需要偏差日志

1. **人脑不是机器**: 人会受到情绪、直觉、外部信息的影响
2. **系统不完美**: 任何量化系统都有局限性
3. **持续改进**: 通过记录偏差，不断优化决策质量
4. **自我认知**: 了解自己的决策模式和能力边界

### 核心概念

```
┌─────────────────────────────────────────────────────────────┐
│                      偏差日志核心流程                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  系统决策 ──▶ 实际决策 ──▶ 偏差识别 ──▶ 记录偏差 ──▶ 定期验证  │
│      │           │           │           │           │     │
│      ▼           ▼           ▼           ▼           ▼     │
│   建议买入    实际卖出    方向相反    偏差日志    结果对比   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 偏差类型详解

### 6种偏差类型

| 偏差类型 | 代码 | 说明 | 典型场景 |
|----------|------|------|----------|
| **人工覆盖** | override | 完全改变系统建议 | 系统建议买入，但您判断风险大而卖出 |
| **延迟执行** | delayed | 推迟执行时机 | 信号出现后等待，错过最佳入场点 |
| **跳过交易** | skipped | 完全忽略信号 | 因资金不足或主观判断而不执行 |
| **修改参数** | modified | 调整数量或价格 | 减少数量、调整买入价位 |
| **额外交易** | additional | 执行计划外交易 | 信号外的主观交易 |

### 偏差严重程度

```
严重 ────────────────────────────────────────▶ 轻微

跳过交易  >  人工覆盖  >  延迟执行  >  修改参数  >  额外交易

影响最大              影响中等              影响较小
```

## 偏差记录流程

### 步骤1: 识别偏差

在录入交易记录时，系统会自动检测是否与关联信号一致：

**一致的情况**:
- 操作方向相同（都买入或都卖出）
- 品种相同

**不一致的情况**（产生偏差）:
- 操作方向相反（系统买，您卖）
- 品种不同
- 跳过了信号未执行
- 执行了信号外的交易

### 步骤2: 填写偏差说明

当系统检测到偏差时，会要求您填写**偏差说明**：

```
┌─────────────────────────────────────────────────┐
│               偏差说明填写指南                     │
├─────────────────────────────────────────────────┤
│                                                 │
│  请说明为什么偏离系统信号：                      │
│                                                 │
│  □ 直觉判断                                     │
│    例如："直觉判断大盘会回调，所以减仓"         │
│                                                 │
│  □ 资金限制                                     │
│    例如："资金被其他品种占用，无法买入"        │
│                                                 │
│  □ 风险控制                                     │
│    例如："当前仓位过重，需要降低风险"          │
│                                                 │
│  □ 信息差异                                     │
│    例如："看到了突发利空消息，紧急卖出"       │
│                                                 │
│  □ 其他原因                                     │
│    请详细说明：____________________             │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 步骤3: 系统自动创建偏差记录

保存交易记录后，系统会自动：

1. **创建偏差日志**: 在`DeviationLog`表中创建记录
2. **设置验证截止**: 默认7天后需要验证
3. **计算偏差类型**: 自动分类偏差类型
4. **发送提醒**: Dashboard显示待验证提醒

## 偏差验证流程

### 什么是偏差验证

偏差验证是在一段时间后（通常7天），**客观评估您的偏离决策是否正确**的过程。

### 为什么要验证

1. **客观评估**: 当时无法判断，时间给出答案
2. **学习改进**: 从成功和失败中学习
3. **能力边界**: 了解自己的决策能力边界
4. **系统优化**: 为系统改进提供数据

### 验证时机

```
偏差发生
    │
    ▼
┌─────────────┐
│  等待期      │ 默认7天
│  (市场演化)  │
└─────────────┘
    │
    ▼
验证截止日期
    │
    ▼
系统提醒验证 ◀── 您收到Dashboard提醒
    │
    ▼
完成验证
```

### 验证步骤

#### 步骤1: 收到提醒

在Dashboard会看到：

```
┌─────────────────────────────────────────────┐
│  ⚠️ 待验证偏差提醒                          │
├─────────────────────────────────────────────┤
│                                             │
│  您有 3 条偏差记录待验证：                   │
│                                             │
│  1. 2026-03-25 跳过买入信号（510300）       │
│     验证截止: 2026-04-01 （今天）           │
│     [立即验证]                              │
│                                             │
│  2. 2026-03-28 提前卖出（510500）           │
│     验证截止: 2026-04-04                    │
│                                             │
│  3. ...                                    │
│                                             │
└─────────────────────────────────────────────┘
```

#### 步骤2: 进入验证页面

点击"立即验证"或进入"偏差日志"页面：

1. 查看偏差详情
2. 对比系统建议与您的决策
3. 查看市场后续走势
4. 评估决策效果

#### 步骤3: 选择验证结果

根据实际情况选择：

**✅ 判断正确**
- 您的偏离决策带来了更好的结果
- 例如：系统建议买入，您判断会下跌而跳过，结果确实下跌

**❌ 判断错误**
- 您的偏离决策导致错失机会或亏损
- 例如：您提前卖出，结果继续上涨

**❓ 无法确定**
- 结果不明显或多种因素交织
- 时间跨度不够，无法客观评估

#### 步骤4: 填写验证笔记

详细记录验证过程和思考：

```
验证笔记示例:
────────────────────────────────
【市场走势】
信号发出后，沪深300指数在接下来3天内下跌了2.5%，
最低触及4.380元。

【决策分析】
当时判断市场短期会调整，所以跳过买入信号。
实际走势验证了判断，避免了约2.5%的浮亏。

【经验总结】
1. 这次判断基于对宏观政策的解读
2. 需要关注类似场景下的判断准确率
3. 不能因此过度自信，仍需严格记录验证
────────────────────────────────
```

#### 步骤5: 保存验证

点击"完成验证"，系统会：

1. 保存验证结果和笔记
2. 计算盈亏差异
3. 更新直觉胜率统计
4. 生成验证报告

## 6. 数据分析

### 6.1 直觉胜率

**直觉胜率 = 验证为"判断正确"的次数 / 总验证次数 × 100%**

Dashboard会显示您的直觉胜率趋势：

```
直觉胜率: 68% (↑ 较上月 +3%)

按偏差类型:
- 人工覆盖: 72%
- 延迟执行: 55%
- 跳过交易: 80%
- 修改参数: 45%
```

### 6.2 盈亏差异分析

系统会统计偏差导致的盈亏差异：

```
累计盈亏差异: +¥2,350 (+1.8%)

说明: 由于您的偏离决策，相比严格跟随系统，
累计多盈利2,350元（收益率高1.8%）

按月统计:
- 2026-01: +¥850
- 2026-02: +¥620
- 2026-03: +¥880
```

### 6.3 决策模式分析

系统会分析您的决策模式：

```
决策模式分析:

优势领域:
✓ 人工覆盖的准确率高（72%）
  → 在关键时点的直觉判断较准
  
✓ 跳过交易的准确率最高（80%）
  → 风险控制方面的判断较好

需要改进:
✗ 修改参数的准确率较低（45%）
  → 调整数量/价格时常判断失误
  → 建议：减少参数修改，严格按信号执行

✗ 延迟执行的准确率偏低（55%）
  → 择时能力有待提高
  → 建议：收到信号后尽快执行
```

## 8. 故障排除

### 8.1 常见问题

**Q: 录入交易时提示"价格超出当日涨跌幅限制"**

**A**: 请检查输入的价格是否正确，是否包含了额外的零。例如4.250写成了42.50。

**Q: 关联信号下拉框为空**

**A**: 说明当前没有待执行的信号。请先在"信号监控"页面确认有活跃信号。

**Q: 保存后找不到交易记录**

**A**: 检查筛选条件是否设置过。点击"重置筛选"查看所有记录。确认交易日期范围。

**Q: 偏差验证时无法选择"判断正确"**

**A**: 请确认偏差记录已过验证期（默认7天）。未到期的偏差无法验证。

### 8.2 数据安全

- **自动保存**: 录入表单时会自动保存草稿
- **数据备份**: 系统每日自动备份数据
- **修改记录**: 所有修改都有日志记录
- **权限控制**: 只能查看和修改自己的交易记录

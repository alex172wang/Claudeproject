# 手动交易操作手册

> **版本**: v1.0  
> **更新日期**: 2026-04-02

---

## 概述

本系统采用**手动操作+系统记录**的模式进行实盘交易。系统负责生成交易信号，您负责在券商APP执行，然后回到系统记录实际交易情况。

### 工作流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│ 系统生成信号 │ ──▶ │ 查看信号    │ ──▶ │ 券商APP执行 │ ──▶ │ 系统记录交易│
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
                                                                   │
                                                                   ▼
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│ 定期复盘验证 │ ◀── │ 统计分析    │ ◀── │ 偏差计算    │ ◀── │ 关联信号    │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
```

## 1. 查看交易信号

### 1.1 信号提醒

当系统生成新的交易信号时，您会在Dashboard看到提醒：

1. **信号卡片**: Dashboard首页的"最新信号时间线"会显示最新信号
2. **提醒标记**: 顶部导航栏会有未读信号提醒
3. **详情页**: 进入"信号监控"页面查看完整信号列表

### 1.2 信号内容

每个信号包含以下信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| 信号时间 | 信号生成时间 | 2026-04-02 14:45:00 |
| 品种 | 建议交易的ETF | 510300 (沪深300ETF) |
| 操作 | 建议的操作 | 买入 |
| 数量 | 建议的股数 | 1000 |
| 综合得分 | 四维评分 | 78.5 |
| 分层得分 | L1-L4各层得分 | L1=85, L2=72, L3=80, L4=65 |

### 1.3 决策参考

在决定是否执行信号时，建议参考：

- **综合得分**: 75分以上为强烈推荐
- **L4风险**: 低于50分表示市场风险较高
- **近期准确率**: 该策略近期执行效果

## 2. 执行交易

### 2.1 券商APP操作

收到信号后，在您常用的券商APP执行交易：

1. **登录券商APP**（如：华泰、中信、招商等）
2. **搜索品种**: 输入ETF代码（如：510300）
3. **确认方向**: 买入/卖出与信号一致
4. **输入数量**: 按信号建议或您自己的判断
5. **确认价格**: 市价或限价委托
6. **提交成交**: 确认委托并等待成交

### 2.2 记录关键信息

在券商APP完成交易后，记录以下信息：

- **成交时间**: 实际成交时间（精确到分钟）
- **成交价格**: 实际成交均价
- **成交数量**: 实际成交股数
- **手续费**: 交易佣金和费用

> **提示**: 可以截图券商APP的成交记录，方便录入系统时参考

## 3. 录入交易记录

### 3.1 进入录入页面

完成交易后，回到本系统录入交易记录：

1. **方式一**: 点击Dashboard顶部"录入交易"按钮
2. **方式二**: 从信号卡片点击"录入执行结果"
3. **方式三**: 进入"交易记录"页面，点击"新建记录"

### 3.2 填写交易信息

在交易录入表单中填写：

#### 必填字段

| 字段 | 说明 | 示例 |
|------|------|------|
| 关联信号 | 选择对应的系统信号（如有） | 信号#12345 |
| 交易时间 | 实际成交时间 | 2026-04-02 14:50 |
| 操作类型 | 买入/卖出/加仓/减仓 | 买入 |
| 品种 | 交易的ETF | 510300 沪深300ETF |
| 数量 | 成交股数 | 1000 |
| 成交价格 | 成交均价 | 4.521 |

#### 可选字段

| 字段 | 说明 | 示例 |
|------|------|------|
| 手续费 | 交易费用 | 5.00 |
| 与信号一致 | 是否与信号完全一致 | 是/否 |
| 偏差说明 | 如不一致，说明原因 | 直觉判断市场会上涨 |
| 交易备注 | 其他备注信息 | 使用融资买入 |

### 3.3 信号匹配标记

**重要**: 请务必正确标记"与信号一致"字段：

- **一致**: 交易品种、方向与信号完全一致（数量可以不同）
- **不一致**: 品种、方向或时机与信号有偏差

如果标记为**不一致**，系统会：
1. 自动创建偏差记录
2. 计算与信号的偏差程度
3. 进入待验证状态

### 3.4 保存记录

填写完成后，点击"保存"按钮：

- 系统会验证表单数据
- 自动计算总金额
- 如与信号不一致，创建偏差记录
- 更新持仓信息
- 刷新交易列表

## 4. 追踪持仓

### 4.1 查看持仓

录入交易后，可以在"持仓管理"页面查看：

- **当前持仓**: 各品种的持仓数量
- **平均成本**: 持仓成本价
- **当前市值**: 按最新价格计算的市值
- **浮动盈亏**: 未实现盈亏

### 4.2 持仓更新

持仓信息会自动更新：

- **买入/加仓**: 增加持仓，重新计算平均成本
- **卖出/减仓**: 减少持仓
- **清仓**: 持仓归零

## 5. 偏差验证

### 5.1 什么是偏差验证

当您执行的交易与系统信号不一致时，系统会创建"偏差记录"。一段时间后（通常1-7天），您可以验证：

- **您的判断是否正确**: 您的操作是否比系统信号更好
- **盈亏差异**: 实际盈亏与系统建议的差异

### 5.2 验证流程

1. **收到验证提醒**: 系统会在Dashboard提醒您有待验证的偏差
2. **查看偏差详情**: 进入"偏差日志"页面，查看待验证的记录
3. **分析市场走势**: 对比验证截止日期的市场情况
4. **完成验证**: 选择验证结果：
   - **判断正确**: 您的偏离带来了更好的结果
   - **判断错误**: 应该跟随系统信号
   - **无法确定**: 结果无明显差异
5. **记录验证笔记**: 说明验证过程和原因

### 5.3 直觉胜率统计

系统会持续统计您的"直觉胜率"：

- **计算方法**: 验证为"判断正确"的次数 / 总验证次数
- **统计维度**: 
  - 总体直觉胜率
  - 按偏差类型（如：人工覆盖、延迟执行等）
  - 按品种
  - 按时间段

### 5.4 使用直觉胜率改进决策

根据直觉胜率，您可以：

- **高胜率（>70%）**: 相信自己的直觉，适当偏离系统
- **中等胜率（40-70%）**: 谨慎偏离，需有充分理由
- **低胜率（<40%）**: 严格跟随系统信号，减少主观干预

## 6. 统计分析

### 6.1 交易统计

系统提供丰富的交易统计：

- **交易频次**: 按日/周/月统计交易次数
- **交易金额**: 统计交易金额分布
- **品种分布**: 各品种的交易占比
- **操作分布**: 买入/卖出/加仓/减仓的占比

### 6.2 信号执行统计

- **执行率**: 信号的执行比例
- **平均响应时间**: 从信号生成到执行的时间
- **完整执行率**: 完全按信号执行的比例
- **偏差率**: 偏离信号的比例

### 6.3 盈亏分析

- **已实现盈亏**: 平仓交易的盈亏
- **未实现盈亏**: 持仓浮动盈亏
- **手续费统计**: 交易成本分析
- **收益率**: 投资收益率计算

## 7. 最佳实践

### 7.1 及时记录

- **立即记录**: 成交后立即录入系统，避免遗忘
- **准确记录**: 确保价格、数量、时间准确
- **详细备注**: 记录决策理由，便于后续分析

### 7.2 诚实标记

- **如实标记**: 如实标记是否与信号一致
- **记录偏差**: 如有偏差，详细说明原因
- **接受验证**: 定期进行偏差验证

### 7.3 定期复盘

- **每日**: 查看当日交易和持仓
- **每周**: 分析交易统计和盈亏
- **每月**: 复盘直觉胜率和偏差验证

### 7.4 持续改进

- **学习**: 从偏差验证中学习
- **调整**: 根据统计数据调整策略
- **优化**: 持续优化交易流程

## 8. 常见问题

### Q1: 如果忘记立即录入交易怎么办？

**A**: 尽快补录，尽量准确回忆交易时间和价格。如有券商APP截图，可参考截图信息。在备注中说明是补录。

### Q2: 部分成交如何记录？

**A**: 可以分多次录入交易记录，每次记录实际成交的部分。系统会自动关联到同一信号。

### Q3: 如何修改已录入的交易？

**A**: 进入"交易记录"页面，找到需要修改的记录，点击"编辑"按钮。修改后保存即可。

### Q4: 什么是"验证截止日期"？

**A**: 这是偏差验证的最后期限。通常在偏差产生后7天，此时市场走势已经明朗，可以进行客观验证。

### Q5: 直觉胜率低怎么办？

**A**: 
1. 减少主观决策，更多跟随系统信号
2. 详细记录每次决策的理由
3. 分析高胜率场景的特点
4. 在低胜率场景强制遵循系统

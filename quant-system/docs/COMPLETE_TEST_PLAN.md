## 四、数据准确性验证标准

### 4.1 价格数据验证
- 价格必须为正数
- 涨跌幅计算准确（change/prev_close * 100）
- 高位 >= 开盘价/收盘价 >= 低位

### 4.2 K线数据验证
- date顺序正确（升序）
- open/high/low/close/volume 无缺失
- 无重复日期

### 4.3 一致性验证
- 同一数据源多次请求结果一致
- 数据库与API返回的数据一致
- 缓存与数据库数据一致（TTL内）
